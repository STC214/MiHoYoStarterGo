# MiHoYoStarterGo

基于 `Go + Wails + Vue3` 的米哈游账号切换与登录辅助工具（本地离线 OCR）。

## 当前状态
- 支持游戏：
  - `GenshinCN`（原神国服）
  - `StarRailCN`（崩坏：星穹铁道国服）
  - `ZZZCN`（绝区零国服）
- 程序可最小化到托盘，支持托盘菜单恢复与双击图标恢复。

## 核心能力
- 本地账号管理（增删改、排序、密码加密存储）
- 游戏路径管理
- 登录流程状态控制（启动、暂停、停止监控）
- OCR 驱动的界面识别与自动登录
- 登录成功后读取注册表 Token 与硬件信息并写回配置

## 原神/崩铁登录流程（最新）
- 账号密码阶段：
  - 识别并填写账号、密码
  - 输入完成后只点击一次 `进入游戏`
- 协议弹窗阶段：
  - 只有在**同一帧 OCR 同时识别到** `不同意` 和 `同意` 时，才允许执行同意点击
  - 点击策略：优先点击“不同意”后紧跟的“同意”；若未命中，回退到同排右侧最近“同意”；再不行则兜底点击最后一个纯“同意”
  - 点击后等待约 1 秒再继续识别
- 成功判定阶段：
  - 识别到 `点击进入` 或 `开始游戏` 视为登录成功
  - 成功后保存注册表 Token 与硬件信息，流程结束

## 绝区零模块（当前实现）
- 沿用 OCR 识别 + 坐标流程。
- 画面 B 支持手动标定以下坐标：
  - 账号输入框
  - 密码输入框
  - 同意协议勾选点
  - 进入游戏点击点
- 坐标按窗口相对位置保存，并按分辨率进行匹配。

## 安全与数据
- 所有识别与自动化流程均在本机执行，不依赖云端 OCR。
- 配置文件位于项目根目录：`config.json`

## 项目结构（主要）
- 后端入口：`app.go`
- 业务层：`app_logic/`
- 核心逻辑：`logic/`
- 前端：`frontend/`

## 本地运行
1. 安装依赖（Go、Node、Wails）
2. 安装前端依赖：
   - `cd frontend`
   - `npm install`
3. 回到项目根目录运行：
   - `wails dev`

## 构建
- 前端构建：
  - `cd frontend`
  - `npm run build`
- Go 编译检查（项目根目录）：
  - `go build ./...`
- Wails 打包（项目根目录）：
  - `wails build`

## 调试说明
- 可通过环境变量开启调试日志：`MHY_DEBUG=1`
- 运行中可在状态弹窗使用“保存调试截图”辅助排查 OCR。
