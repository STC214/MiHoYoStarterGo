# MiHoYoStarterGo

基于 `Go + Wails + Vue3` 的米哈游账号切换与登录辅助工具（本地离线 OCR）。

## 当前状态
- 支持游戏：
  - `GenshinCN`（原神国服）
  - `StarRailCN`（崩坏：星穹铁道）
  - `ZZZCN`（绝区零）
- 程序可最小化到托盘，支持：
  - 托盘菜单恢复主界面
  - 双击托盘图标恢复主界面

## 核心能力
- 本地账号管理（增删改、排序、密码加密存储）
- 游戏路径管理
- 登录流程状态控制（启动、暂停、停止监控）
- OCR 驱动的界面识别与自动登录
- 登录成功后读取注册表 Token 与硬件信息并写回配置

## 绝区零模块（当前实现）
- 识别策略沿用崩铁 OCR 方案，并增加模糊匹配能力。
- 登录流程按画面分阶段：
  - 画面 A：识别 `+86`、`账号密码`，自动点击“账号密码”切换
  - 画面 B：识别 `输入手机号/手机号`、`邮箱`、`用户` 后，进入坐标登录流程
  - 画面 C：在窗口下方约五分之一区域识别 `点击进入`，视为登录成功
- 画面 B 的实际输入/点击使用“手动标定坐标”：
  - 账号输入框
  - 密码输入框
  - 同意协议勾选点
  - 进入游戏点击点
- 标定方式为倒计时自动采样鼠标位置（不依赖热键/点击事件捕获）。
- 坐标按游戏窗口相对坐标保存，并按分辨率匹配（含近似匹配兜底）。

## 安全与数据
- 所有识别与流程均在本机执行，不依赖云端 OCR 服务。
- 配置文件位于项目根目录：`config.json`
- 调试截图输出到项目根目录。

## 项目结构（主要）
- 后端入口：`app.go`
- 业务层：`app_logic/`
- 核心逻辑：`logic/`
- 前端：`frontend/`

## 本地运行
1. 安装依赖（Go、Node、Wails）
2. 安装前端依赖：
   - `cd frontend`
   - `npm install`
3. 回到项目根目录运行：
   - `wails dev`

## 构建
- 前端构建：
  - `cd frontend`
  - `npm run build`
- Go 编译检查（项目根目录）：
  - `go build ./...`
- Wails 打包（项目根目录）：
  - `wails build`

## 调试说明
- 可通过环境变量开启调试日志：
  - `MHY_DEBUG=1`
- 运行中可在状态弹窗使用“保存调试截图”辅助排查 OCR。
